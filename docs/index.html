§ 01

Why This Exists

Defense supply chains increasingly depend on distributed additive manufacturing. A part printed in a contractor's facility for a fielded system carries an implicit trust assumption: that it was printed correctly, with the right material, under the right thermal conditions, by a node whose firmware hasn't been tampered with.

There is currently no cryptographic protocol for that assumption. axm-sfn is the protocol.

The Custody Clock

The 1 Hz custody ticker is independent of Moonraker's event stream. Printer silence is recorded, not ignored. If the machine goes quiet mid-print, the custody clock keeps ticking and records a provenance fault. You cannot delete history by disconnecting the sensor.

Hardware Binding

Every custody packet is signed by a TPM 2.0 RSA-PSS key that never leaves the node. TPM quotes bind the PCR state of the platform to specific points in the telemetry chain. A VM cannot impersonate a physical fabrication node.

Non-Selective Recording

AXM REQ 5 is enforced at the protocol level. The cam_latents.bin frame sequence must be gap-free and monotone. Any discontinuity is a hard verification failure — the same exit code as a forged signature. A node that drops frames is not conformant.

The Tollbooth Model

The protocol is open. Anyone can run axm-edge. Certification is the barrier to entry. To appear on an approved vendor list, a node must pass the axm-sfn conformance suite, submit a NIST challenge artifact, and produce a signed Material-Process Profile shard. axm-verify is the tollbooth.

§ 02

Daemon Pipeline

axm-edge is a Go daemon that runs alongside Klipper on the printer's host (Raspberry Pi 4 or equivalent). It coordinates four subsystems — Moonraker state, custody timing, TPM attestation, and the SQLite WAL hot buffer — into a single tamper-evident telemetry chain.

Moonraker WS
internal/moonraker
JSON-RPC WebSocket client. Subscribes to extruder, bed, motion, print_stats, MCU, and optional chamber sensor. Emits StatusDelta on state changes. Auto-reconnects. Never blocks the Klipper motion path.
▼   state deltas → State Cache (sync.RWMutex)   ▼   1Hz tick (independent of Moonraker)
Custody Packetizer
internal/custody
The custody clock. Fires every 1s regardless of Moonraker activity. Snapshots the state cache, evaluates the active Material-Process Profile via policy.Evaluator, calls tpm.SignPacket(), and computes the BLAKE3 chain link.
▼   PacketRow → SQLite WAL (survives network outage, power loss recovery on next boot)
Hot Buffer
internal/hotbuffer
SQLite WAL. Append-only during print. Stores telemetry JSON, TPM sig, BLAKE3 chain, quote references, and segment upload state. A 40-hour print survives a network outage intact. Gaps recorded as provenance faults.
▼   on retryInterval: PacketHashes() → BLAKE3 Merkle root → WriteSegment()
Uploader
internal/uploader
Batches contiguous pending packets into Merkle segments (default: 60 packets = 1 minute). Computes the RFC 6962 BLAKE3 Merkle root. POSTs to NodalFlow. Falls back to stdout JSON in dev mode if no endpoint is configured.
▼   segment → NodalFlow → Forge → Genesis → AXM Layer 2 journal shard (ML-DSA-44 signed)
AXM Layer 2
axm-genesis
Journal shards sealed by Genesis. Full decision trace: what MPF was active, what tolerances were evaluated, what thresholds were crossed, what was certified. Verifiable by any party with the publisher key, decades later.

§ 04

AXLF / AXLR Binary Format

The fabrication hot stream is the same format as axm-embodied — file cam_latents.bin, file magic AXLF, record magic AXLR. Because it adopts the embodied format verbatim, the frozen Genesis kernel validates REQ 5 frame continuity on fabrication shards with zero changes and no spoke-specific verifier. The spoke differs only in what the 256-byte payload means, not in the container or the continuity guarantee.

Offset Math

Record offset for frame N: offset(N) = 4 + N × 269  (4-byte file header + N × 13-byte record header + 256-byte payload). Records are scanned linearly. No index. No random access. Any gap in the monotone frame_id sequence triggers verification failure.

§ 05

Material-Process Profile Engine

The internal/policy engine evaluates each 1 Hz custody tick against a versioned Material-Process Profile (MPF). Rules map telemetry field names to mathematical envelopes. If no profile is loaded, the null policy passes everything — the daemon runs on any node.

schema/mpf/mat-asa-prosumer-v1.0.json — example MPFjson
{
  "profile_id": "mat-asa-prosumer-v1.0",
  "validation_rules": [
    {
      "field": "extruder.temperature",
      "target_field": "extruder.target",
      "rule_type": "dynamic_offset",        // actual must stay within ±ΔT of target
      "max_negative_deviation": 5.0,
      "max_positive_deviation": 10.0,
      "allowed_consecutive_violation_epochs": 3
    },
    {
      "field": "temperature_sensor.chamber.temperature",
      "rule_type": "absolute_minimum",
      "min_value": 90.0,                         // ASA requires enclosure ≥ 90°C
      "allowed_total_violation_epochs": 15
    }
  ]
}
Design: Flat Float Map

The packetizer extracts telemetry into a map[string]float64 once per tick. No JSON Pointer parsing in the hot path. The policy engine iterates the flat map against the active profile's rules. Absent fields are skipped silently — printers without a chamber sensor pass chamber rules without error. Chamber temp is only added to the map when ChamberPresent=true, set by the first Moonraker delta that reports it.

Quick Start

Build + Run
# Clone and build
$ git clone https://github.com/BigBirdReturns/axm-sfn
$ cd axm-sfn && go mod tidy && go build ./cmd/axm-edge/
 
# Provision TPM keys (run once — requires tss group membership)
$ sudo ./axm-edge provision --config config.example.yaml

axm-edge: persistent keys verified sign=0x81000001 ak=0x81000002
 
# Run in dev mode (stdout segments, no NodalFlow required)
$ ./axm-edge run --config config.example.yaml

TPM Not Required for Development

If /dev/tpmrm0 is absent, axm-edge runs in software-only mode. Packets are hashed and chained but not TPM-signed. The hot buffer and custody loop function normally. The daemon logs TPM unavailable — running without hardware attestation and continues. This makes development on any Linux machine possible without a physical TPM.
